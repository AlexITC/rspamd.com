Performance

Rspamd is optimized to work as fast as possible. There is numerous of optimisations that allow your mail to be processed faster. Rspamd uses event based processing model and do not block anywhere in the code. Moreover, it is possible to use multiply threads for paralleling complex tasks and multiply processes to serve requests. This allows the processing of hundreds messages per second without a significant CPU load.

Features

Rspamd supports various internal and external filters to process messages, such as regular expressions, suffix tries, RBL, URL black lists, IP lists, SPF, DKIM, character maps. For advanced filtering rspamd provides improved statistcs module (based on OSB-Bayes algorithm) and fuzzy hashes database that is generated based on honeypots traffic.

Extensions

Rspamd can easily be extended with your own rules and plugins that could be written in lua language. Moreover, rspamd ships with numerous of lua plugins and rules. The extensive lua API documentation is also available.
